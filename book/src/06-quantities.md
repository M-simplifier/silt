# 6. 数量: `0`、`1`、`omega`

Siltのbinderには数量があります。

数量は、「この値を何回使うか」を型の近くに置くための仕組みです。普通のプログラムでは細かすぎるように見えるかもしれません。しかし、低レイヤーでは、値を捨てたのか、消費したのか、実行時には残さないのかが重要になります。

使える数量は三つです。

| 数量 | 意味 |
|---|---|
| `0` | 実行時には使わない。型や証明のための引数 |
| `1` | ちょうど一度使う線形値 |
| `omega` | 何度でも使える通常の値 |

数量を省略すると `omega` です。

```lisp
(fn ((x U64)) x)
```

これは `x` を何度でも使える値として受け取ります。

型引数はよく `0` になります。

```lisp
(claim id (Pi ((A 0 Type) (x A)) A))
(def id
  (fn ((A 0 Type) (x A)) x))
```

`A` は型検査に必要ですが、生成されるC関数の実行時引数には不要です。SiltのC backendは、`0` 引数を実行時シグネチャから消します。

ここで重要なのは、`0` が「どうでもよい」という意味ではないことです。むしろ逆です。型検査には重要だが、実行時には持ち込まない、という区別を表しています。

`1` は、リソースやeffectの結果を扱うときに重要です。

```lisp
(bind SerialReady SerialReady SerialReady U64 Unit serial-line-status
  (fn ((status 1 U64))
    ...))
```

`status` は `serial-line-status` の結果です。後続の処理で一度だけ使われます。Siltの数量チェックは完全な線形型システムではありませんが、値の重複利用や消費忘れを早い段階で検出するための土台になっています。

使わない結果は、線形binderで受け取ってから `let` で消します。

```lisp
(fn ((stored 1 Unit))
  (let ((ignored-stored 0 stored))
    next-action))
```

これは少し回り道に見えますが、重要な習慣です。effectfulな操作は必ず実行されるべきです。しかし、その戻り値 `Unit` は後続で使わないことがあります。そこで、結果を一度受け取り、`0` の束縛で「値としては使わない」と明示します。

Siltの数量は、リソース、capability、証明消去を扱うための入口です。値を通常のデータとして使うのか、一度だけ消費するものとして使うのか、型や証明のためだけに使うのかを、binderの時点で分けます。

数量を読むと、プログラムの表面に「値の寿命」が見え始めます。これは、あとで静的storageやallocator handoffを読むときにも効きます。
