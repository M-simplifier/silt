# この本の読み方

Siltを読むときは、コードを「実行される命令」としてだけ見ないでください。型、数量、effect state、layout、セクション、ABI、QEMU markerが、それぞれ別の層で同じプログラムを支えています。

この本は、その層を順番に読むためのものです。

この本では次の順序で進みます。

1. S式と依存型コアを読む
2. 数量とeffect stateで「値をどう使ったか」「副作用がどこへ進むか」を読む
3. `layout`、ポインタ、静的オブジェクトでメモリ表現へ降りる
4. ABI、freestanding C、Limine、QEMUで低レイヤー実行物へつなぐ
5. allocator handoffを、ひとつの総合例として読む

各章には小さなコード片を置きます。多くのコードは実際の `examples/` に由来しますが、説明のために短くしたものもあります。まず小さな形で読み、あとで完全な実例へ戻るのが、この本の進め方です。完全な実例は `examples/limine.silt`、`examples/limine-serial.silt`、`examples/limine-protocol.silt` にあります。

この本で使う言葉には、少し注意が必要です。

`純粋` とは、何も副作用を書かないという意味ではありません。Siltでは副作用を `Eff pre post A` として型に載せます。シリアルポートへの `outb` も、メモリからの `load` も、型付きの状態遷移として書きます。

`低レイヤー` とは、Cの構文に似せることではありません。サイズ、アラインメント、フィールドオフセット、セクション、ABI、ブートプロトコル、I/Oポートのような事実を、Siltの型付き構文で明示することです。

`検証` とは、すべてが形式証明済みという意味ではありません。Siltでは、型検査、正規化、生成C、ELF/linker、Limine config、QEMU serial markerを組み合わせて、実装済みの範囲を検証しています。

この本は順番に読む構成ですが、すでに低レイヤーに慣れている読者はPart IIIから読んでも構いません。依存型に慣れている読者はPart IIを軽く流し、Part IVでSiltらしさを確認するとよいでしょう。

どこから読んでも、同じ問いに戻ってきます。この値は、どの層で守られているのか。この操作は、どの状態からどの状態へ進むのか。この表現は、機械の上でどの形になるのか。Siltの学習は、その問いを自分で追えるようになることです。
