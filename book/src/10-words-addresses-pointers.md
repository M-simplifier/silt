# 10. `U8`、`U64`、`Addr`、`Ptr`

低レイヤーを書くには、機械語に近い値が必要です。

ただし、Siltではそれらを裸の整数としてだけ扱いません。バイト、word、アドレス、ポインタを分け、それぞれに型を与えます。Siltには、次のruntime-backed primitiveがあります。

- `U8`
- `U64`
- `Addr`
- `Ptr A`

`U64` リテラルは次のように書きます。

```lisp
(u64 4096)
```

`U8` は `0..255` の範囲でチェックされます。

```lisp
(u8 83)
```

`Addr` はアドレス値です。

```lisp
(addr 1048576)
```

ポインタは `Ptr A` です。

```lisp
(Ptr U64)
(Ptr LimineHhdmResponse)
```

ポインタは、アドレスへ変換できます。

```lisp
(ptr-to-addr U64 ptr)
```

アドレスからポインタを作ることもできます。

```lisp
(ptr-from-addr U64 (addr 0))
```

これは低レイヤーでは必要ですが、危険な操作です。Siltは完全なaliasing/ownership検証を提供しません。そのため、この操作は「低レイヤー表現の明示」として使い、広い安全性を主張しすぎないことが重要です。

ここで得られるのは、魔法の安全性ではありません。`addr` から `Ptr A` へ変換した時点で、プログラムは「このアドレスをAへのポインタとして読む」という意図を明示します。その意図が正しいかどうかは、周辺のlayout、boot情報、検証経路と合わせて判断します。

サイズとアラインメントは `size-of` と `align-of` で取得できます。

```lisp
(size-of U64)
(align-of U64)
```

layout型にも使えます。

```lisp
(size-of BootInfo)
(align-of BootInfo)
```

`Ptr A` はC backendでは実行時に通常のアドレス値として扱われますが、Siltの型では「何を指すポインタか」を保持します。`Ptr U64` と `Ptr BootInfo` は同じ機械表現を持っていても、同じ型ではありません。

この区別が、Siltで低レイヤーを書くときの基本姿勢です。機械に近い表現を使いながら、表面では型付きの意図をできるだけ残します。

低レイヤーでは、すべてを一つの整数として見たくなる場面が何度もあります。Siltはそこで、少しだけ立ち止まらせます。その値はbyteなのか、wordなのか、アドレスなのか、何を指すポインタなのか。その問いが、次の `layout` へつながります。
