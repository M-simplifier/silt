# 5. `Pi`と関数

Siltの関数型は `Pi` で書きます。

最初は、`Pi` を「少し情報量の多い関数型」と読めば十分です。

```lisp
(Pi ((x A)) B)
```

これは、`A` 型の `x` を受け取り、`B` 型を返す関数型です。戻り値の型 `B` は `x` に依存できます。依存しない場合は、普通の関数型と同じように読めます。

複数引数は次のように書きます。

```lisp
(Pi ((x U64) (y U64)) U64)
```

関数本体は `fn` です。

```lisp
(claim add
  (Pi ((x U64) (y U64)) U64))
(def add
  (fn ((x U64) (y U64))
    (u64-add x y)))
```

Siltでは、引数には数量を付けられます。

```lisp
(Pi ((A 0 Type) (x A)) A)
```

ここで `A 0 Type` は、`A` が実行時には使われない型引数であることを示します。数量を省略すると `omega`、つまり何度でも使える通常の値として扱われます。

`Pi` は依存型の入口ですが、低レイヤーにも関係します。たとえば `qemu-debug-exit` は次の型を持ちます。

```lisp
(claim qemu-debug-exit
  (Pi ((pre 0 Type)
       (post 0 Type)
       (code U64))
      (Eff pre post Unit)))
```

`pre` と `post` は実行時には消える型レベル引数です。しかし、型の中では副作用の状態遷移を表しています。

この関数は、「任意の状態 `pre` から任意の状態 `post` へ進むdebug-exit操作」として読めます。実装は `x86-out8` です。

```lisp
(def qemu-debug-exit
  (fn ((pre 0 Type)
       (post 0 Type)
       (code U64))
    (x86-out8 pre post (u64 244) code)))
```

このように、`Pi` は抽象的な数学のためだけにあるのではありません。低レイヤー操作の「どの状態からどの状態へ進むか」を型に入れるためにも使われます。

Siltで `Pi` を読むときは、二つの問いを持つとよいです。どの引数が実行時に残るのか。どの引数が、型の中で約束を表すためだけに使われるのか。この二つを分けられると、数量の章へ自然につながります。
